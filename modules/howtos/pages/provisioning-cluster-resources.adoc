The primary means for managing clusters is through the Couchbase Web UI which provides an easy to use interface for adding, removing, monitoring and modifying buckets. 
Or through the REST API.
In some instances you may wish to have a programmatic interface. For example, if you wish to manage a cluster from a setup script, or if you are setting up buckets in test scaffolding.






The {name-sdk} also comes with some convenience functionality for common Couchbase management requests.

Management operations in the SDK may be performed through several interfaces depending on the object:

* BucketManager -- `Couchbase::Management::BucketManager`
-- see https://docs.couchbase.com/sdk-api/couchbase-cxx-client-1.0.0-dp.9/classcouchbase_1_1bucket__manager.html[API docs].
